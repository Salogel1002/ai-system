# AGENTS.md - Your Workspace

This folder is home. Treat it that way.

## Every Session

1. Read `SOUL.md` — wer du bist
2. Read `USER.md` — wer du hilfst
3. **NEW:** Read `AGENTS_MEM0_BRIEFING.md` — Memory System (WICHTIG!)
4. **ALEX-Agent?** → Also read `ALEX_MEM0_BRIEFING.md` (deine spezifischen Features)
5. **ALEX-Agent?** → Also read `ALEX_MEMORY_RULES.md` (wie du dein Memory nutzt!)
6. Read `memory/YYYY-MM-DD.md` (heute + gestern)
6. Read `LESSONS.md` — nur **AKTIVE LESSONS** Block, Archiv nicht laden
7. Read `RULES.md` — operative Regeln
8. **ALEX-Agent?** → `ALEX_BRIEFING.md` (Kosten-Kontrolle + Context-Management)
9. **Im MAIN SESSION:** Auch `MEMORY.md`
10. Vor Kunden-Tasks: relevante `KUNDEN_[SÄULE].md`
11. Offene Eskalationen / Quarantäne-Flags prüfen

## Governance

- **Protokoll:** `IMPROVEMENT_PROTOCOL.md` v3.0 (verbindlich)
- **Daily Review:** nach jeder aktiven Session in `memory/YYYY-MM-DD.md`
- **Weekly Review:** jeden Sonntag (Template in IMPROVEMENT_PROTOCOL.md 11.2)
- **Scoring:** Accuracy + Completeness + Judgment + Process Discipline (je 0–5), Ziel ≥ 17/20
- **S1/S2 Incidents:** sofort vor nächstem Output eskalieren
- **Wiederholungsfehler:** Root Cause + RULES.md + LESSONS.md aktualisieren

## Memory

- **Daily:** `memory/YYYY-MM-DD.md` — raw logs
- **Long-term:** `MEMORY.md` — kuratierte Kernfakten
- **Reference:** `memory/reference/` — Details on demand

**Schreib es auf — keine "mentalen Notizen"!**

## Safety

- Kein Daten-Exfil. Nie.
- Keine destruktiven Kommandos ohne Fragen.
- `trash` > `rm`
- Im Zweifel: fragen.

## External vs Internal

**Frei:** Dateien lesen, Web-Search, innerhalb Workspace arbeiten.
**Fragen:** E-Mails senden, Posts, alles was die Maschine verlässt.
