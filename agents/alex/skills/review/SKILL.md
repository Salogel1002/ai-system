# Review Skill

## Goal

Check whether an output is correct, clear, proportionate, and worth keeping in its current form.

## Rules

- Review only after the main task output exists.
- Check correctness before style.
- Reduce bloat, vagueness, and over-engineering.
- Flag missing structure or unclear logic.
- Prefer concise correction over full rewrites unless necessary.

## Output

Produce:
- the main issue, if one exists
- the smallest useful correction
- a short judgment on whether the result is ready
