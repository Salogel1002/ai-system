# Skill Intake Flow

Der Skill Intake Flow beschreibt den kontrollierten Aufnahmeprozess für neue Skills in das System. Er gilt für interne, externe, Partner- und über Clawhub installierte Skills.

## Zweck

Dieses Dokument stellt sicher, dass neue Skills nicht direkt produktiv nutzbar werden, sondern einen formalen Aufnahmeprozess durchlaufen.

Ohne Skill Intake Flow entstehen:
- unkontrollierte Skill-Zugänge
- unklare Verantwortlichkeiten
- fehlende Sicherheits- und Qualitätsprüfung
- Chaos durch direkt eingebrachte externe Skills

## Grundprinzipien

### 1. Installation ist nicht Aktivierung
Ein Skill darf nach Installation oder Bereitstellung nicht automatisch verwendet werden.

### 2. Intake vor Nutzung
Jeder neue Skill muss zuerst formal aufgenommen werden.

### 3. Einheitlicher Prozess für alle Quellen
Interne und externe Skills folgen demselben Governance-Grundmuster, auch wenn die Prüftiefe unterschiedlich sein kann.

### 4. Dokumentierter Statuswechsel
Jeder Schritt im Intake-Prozess muss nachvollziehbar sein.
